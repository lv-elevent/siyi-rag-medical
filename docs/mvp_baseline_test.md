# RAG MVP 基线验证流程

本文档记录当前 RAG MVP 的基线验证流程，用于确保每次代码修改后系统功能正常。

---

## 1. 项目启动前检查项

### 1.1 环境依赖检查
- [ ] Python 虚拟环境已激活（`.venv` 目录存在）
- [ ] 已安装所有依赖包（运行 `pip install -r requirements.txt`）
- [ ] 已安装 PyMuPDF（用于 PDF 解析）
- [ ] 已安装 ChromaDB（用于向量数据库）
- [ ] 已安装 OpenAI 兼容的 LLM 客户端

### 1.2 配置文件检查
- [ ] `backend/core/config.py` 配置正确
- [ ] LLM API Key 已配置
- [ ] 向量数据库路径配置正确
- [ ] 上传目录 `uploads/` 已创建

### 1.3 目录结构检查
```
d:\rag-knowledgebase\
├── backend/
│   ├── api/           # API 路由
│   ├── core/          # 核心配置
│   ├── models/        # 数据模型
│   ├── repositories/  # 数据访问层
│   ├── services/      # 业务逻辑
│   └── main.py        # 主应用
├── frontend/          # 前端代码
├── uploads/           # PDF 上传目录（需手动创建）
└── docs/              # 文档目录
```

### 1.4 端口占用检查
- [ ] 8000 端口未被占用（后端）
- [ ] 端口 8000 可正常访问

## 2. 启动命令

### 2.1 启动后端服务
```bash
# 进入项目根目录
cd d:\rag-knowledgebase

# 激活虚拟环境
.venv\Scripts\activate

# 启动后端
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

### 2.2 启动前端服务
```bash
# 打开新的终端窗口

# 进入前端目录
cd d:\rag-knowledgebase\frontend

# 启动前端（使用 Python 服务器）
python -m http.server 3000
```

### 2.3 同时启动（推荐）
```bash
# 终端 1 - 启动后端
cd d:\rag-knowledgebase
.venv\Scripts\activate
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000

# 终端 2 - 启动前端
cd d:\rag-knowledgebase\frontend
python -m http.server 3000
```

### 2.4 访问应用
- **后端 API**：http://127.0.0.1:8000
- **API 文档**：http://127.0.0.1:8000/docs
- **前端界面**：http://127.0.0.1:3000

## 3. 上传 PDF 后的验证步骤

### 3.1 上传测试文件
1. 打开前端界面：http://127.0.0.1:3000
2. 点击上传区域，选择一个测试 PDF 文件
3. 等待上传完成，观察状态提示："文件 xxx 上传成功"

### 3.2 检查知识库状态
1. 查看左侧"知识库状态"区域
2. 应该显示：
   - 当前文档：[上传的文件名]
   - 状态：已加载，可进行问答
3. 检查后端控制台，应该看到：
   - `[upload] PDF 解析成功: xxx.pdf`
   - `[upload] 提取文本长度: xxx`
   - `[upload] 文本分块完成: chunk 数量 = xxx`
   - `[upload] Embedding 生成完成: 数量 = xxx`
   - `[upload] ChromaDB 入库完成`
   - `[upload] 已清空旧知识库`

### 3.3 验证文件保存
1. 检查 `uploads/` 目录是否存在
2. 应该能看到上传的 PDF 文件

### 3.4 验证向量数据库
1. 检查 ChromaDB 是否正常工作
2. 应该能看到向量集合已创建
3. 检查后端控制台，应该看到向量数量与 chunk 数量一致

## 5. 每次修改后必须回归验证

### 5.1 修改范围清单

#### 后端修改必须验证：
- [ ] `/upload` 接口：PDF 上传、保存、解析、分块、Embedding、向量入库
- [ ] `/knowledge/status` 接口：知识库状态查询
- [ ] `/chat` 接口：问答功能（检索 + LLM 生成）
- [ ] 文件保存功能：`uploads/` 目录、文件权限
- [ ] PDF 解析功能：`backend/services/pdf_parser.py`
- [ ] 文本分块功能：`backend/services/text_chunker.py`
- [ ] Embedding 生成功能：`backend/core/embedding_client.py`
- [ ] 向量数据库操作：`backend/repositories/vector_repository.py`
- [ ] LLM 调用功能：`backend/core/llm_client.py`

#### 前端修改必须验证：
- [ ] 文件上传功能：选择、上传、状态显示
- [ ] 知识库状态显示：空状态、处理中、就绪状态
- [ ] 聊天功能：发送消息、显示 AI 回答
- [ ] 错误处理：上传失败、提问失败

#### 配置修改必须验证：
- [ ] 环境变量配置：LLM API Key、数据库路径
- [ ] 端口配置：后端、前端
- [ ] 上传目录配置：`uploads/` 路径

### 5.2 回归验证步骤

1. **启动验证**
   - [ ] 后端正常启动，无错误日志
   - [ ] 前端正常启动，页面可访问
   - [ ] API 文档可访问：http://127.0.0.1:8000/docs

2. **上传验证**
   - [ ] 能成功上传 PDF 文件
   - [ ] 上传后状态显示正确
   - [ ] 文件正确保存到 `uploads/` 目录
   - [ ] 知识库状态显示为"已加载，可进行问答"

3. **问答验证**
   - [ ] 能正常发送问题
   - [ ] 能收到 AI 回答
   - [ ] 固定测试问题得到正确回答

4. **错误处理验证**
   - [ ] 上传非 PDF 文件时正确提示
   - [ ] 未上传文件时无法发送问题
   - [ ] 网络错误时正确提示

### 5.3 验证通过标准

所有验证项都打勾后，认为本次修改通过回归验证。如果任何一项未通过，需要：
1. 记录问题
2. 修复问题
3. 重新验证

## 6. 常见问题排查

### 6.1 后端启动失败
**问题**：端口 8000 已被占用
**解决**：检查并关闭占用端口的进程

### 6.2 PDF 解析失败
**问题**：后端控制台显示 PDF 解析错误
**解决**：
- 检查 PDF 文件是否损坏
- 检查 `PyMuPDF` 是否正确安装

### 6.3 向量数据库错误
**问题**：向量入库失败
**解决**：
- 检查 ChromaDB 是否正确安装
- 检查 `uploads/` 目录是否有写入权限

### 6.4 LLM 调用失败
**问题**：问答功能返回错误
**解决**：
- 检查 LLM API Key 是否正确配置
- 检查网络连接是否正常
