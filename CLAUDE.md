# Claude Code 任务分析与规划文档

## 项目背景

我们正在开发一个 **RAG 知识库问答系统**，当前已经完成中后期优化。系统基于 **FastAPI**，使用 **ChromaDB** 作为向量数据库，并通过前端实现文件上传、问答展示等功能。现阶段，系统已经具备了以下功能：

### 已完成的功能
1. **Embedding 统一与归一化**：
   - 模型：`BAAI/bge-large-zh-v1.5`
   - 向量 norm ≈ 1
   - 距离恢复正常：0.6 ~ 0.7

2. **Embedding 缓存**：
   - 避免重复调用 API

3. **动态 Top-K 调整**：
   - 根据知识库大小动态选择 top_k。

4. **检索优化**：
   - 包括检索链路：candidate retrieval → dynamic distance filter → dedup → rerank → LLM

5. **动态过滤策略**：
   - 根据问题长度调整 margin（短问题：0.08，中问题：0.10，长问题：0.12）

6. **轻量 rerank**：
   - 基于 distance、keyword、length。

7. **文档管理**：
   - 使用 SHA256 去重，提供删除功能。

8. **用户体验优化**：
   - 支持无上传问答，文档选择高亮，紧凑布局。

## 第6阶段目标

**医疗RAG系统升级（第6阶段）**

在此阶段，我们需要将当前的系统升级为 **医疗级RAG系统**，使其能够处理复杂的医疗相关问答任务，具备以下能力：

- 医疗问句识别
- 医疗问句改写（Query Rewrite）
- 上下文安全过滤
- 医疗风格回答结构
- 多文档融合（可选）
- 引入跨文档精排（Cross-encoder）

## Claude Code 任务分析与规划

### 1. 系统现状分析

分析现有系统架构和能力，识别当前的优势和局限性：

- **架构分析**：评估 FastAPI 和 ChromaDB 的使用是否合理，当前的数据结构是否适合医疗问答的需求。
- **功能分析**：各模块的功能是否已经完全符合医疗领域的特殊要求（如安全性、准确性）。
- **能力分析**：当前的检索性能、问答精度、系统的响应速度是否达到医疗级要求。

### 2. 第6阶段任务规划

以下是医疗RAG系统升级的任务和开发方案。

#### 任务1：医疗问句识别（Query Classification）

**目标**：识别用户输入的问句类型，输出三种类型之一：

- "medical"：医疗问诊类（症状、疾病、处理建议等）
- "knowledge"：医疗知识类（概念、定义、机制等）
- "other"：非医疗问题

**开发方案**：
- 使用简单的文本分类方法，如基于关键词的匹配、正则表达式或者小型模型来判断问句类型。
- 在 `chat.py` 或 `retrieval_service.py` 中添加分类逻辑。

**验证方法**：
- 使用一些典型的医疗问句和非医疗问句进行测试，确保分类准确。

#### 任务2：医疗检索增强（Query Rewrite）

**目标**：将口语化的医疗问句改写为更具检索价值的详细问句。

**开发方案**：
- 对医疗类问句进行改写，确保改写后的问句能够有效检索到相关文档。
- 在 `retrieval_service.py` 中增加改写逻辑，可以使用简单的模板替换或基于模型的文本生成。

**验证方法**：
- 测试改写前后的检索结果，确保改写后的问句能提升检索效果。

#### 任务3：上下文安全过滤

**目标**：确保系统输出不包含危险的医疗建议，如药物剂量错误、误诊等。

**开发方案**：
- 在生成的回答中检测是否含有绝对性的诊断或药量推荐。
- 如果有危险内容，替换为适当的免责声明或建议就医。

**验证方法**：
- 测试系统输出，确保不出现药物剂量、误诊等问题。

#### 任务4：回答结构升级

**目标**：将问答输出升级为符合医疗领域标准的结构，包括【可能原因】【建议措施】【需要就医情况】【免责声明】。

**开发方案**：
- 在生成答案时，按照医疗风格组织内容。
- 使用模板确保回答结构一致。

**验证方法**：
- 验证输出格式是否符合要求，并测试输出内容的医疗准确性。

#### 任务5：多文档融合（可选）

**目标**：从多个文档中提取信息并融合生成完整的回答。

**开发方案**：
- 扩展检索结果，融合来自多个文档的信息，提高回答的全面性。

**验证方法**：
- 使用多个文档进行测试，确保信息融合合理。

#### 任务6：跨文档精排（Cross-encoder）

**目标**：引入 Cross-encoder 进行候选文档的精细排序，提高检索精度。

**开发方案**：
- 在 `retrieval_service.py` 中使用 Cross-encoder 对候选文档进行精排，输出最相关文档。

**验证方法**：
- 测试 Cross-encoder 排序后的结果，确保检索结果更为相关和精准。

## 开发优先级与依赖关系

1. **医疗问句识别** → 必须首先完成，后续任务依赖此任务的分类结果。
2. **医疗检索增强** → 紧跟在问句识别后，改写逻辑能提高检索精度。
3. **上下文安全过滤** → 一旦开始生成医疗回答，安全过滤是必须的。
4. **回答结构升级** → 安全过滤后，需要将输出内容组织成标准格式。
5. **多文档融合** 和 **跨文档精排** → 可选任务，优先级较低，但能够进一步提升系统性能。
